# 中文填表助手

一个面向中文场景的 Chrome 扩展，用来把网页表单快速填充为更真实的中文虚拟数据，适合本地联调、表单验证、演示环境和组件库适配测试。

## 快速开始

如果你只是想尽快跑起来，可以按下面的最短路径操作：

```bash
pnpm install
pnpm build
```

然后在 Chrome 中：

1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择项目中的 `dist/` 目录

完成后打开任意包含表单的网页，点击扩展图标或使用快捷键即可测试。

## 功能特性

- 一键填充当前页面中的表单字段
- 支持只填充当前表单或当前输入框
- 支持工具栏点击、快捷键、右键菜单三种触发方式
- 内置中文姓名、手机号、邮箱、地址、公司、身份证号、日期等常见数据生成逻辑
- 支持配置字段匹配规则，可按 `name`、`id`、`label`、`placeholder`、`aria-label` 识别字段
- 支持忽略隐藏字段、跳过已有内容字段、触发输入事件
- 支持固定密码或随机密码策略
- 支持自定义字段规则，为特定关键词指定填充值类型
- 提供本地 playground，便于验证对组件库表单控件的适配效果

## 支持范围

当前项目主要覆盖以下场景：

- 原生 HTML 表单控件
- Element Plus 2.x 的部分表单组件
- Ant Design Vue 5.x 的部分表单组件

说明：

- 组件库适配采用针对性选择器和交互逻辑实现，版本升级后可能需要同步调整。
- 对高度定制、强业务绑定或封装较深的表单控件，可能需要额外补充匹配规则或适配逻辑。

## 触发方式

### 1. 点击扩展图标

点击 Chrome 工具栏中的扩展图标，向当前页面发送“填充全部”指令。

### 2. 快捷键

默认快捷键如下：

- `Ctrl + Shift + F`：填充所有输入框
- `Ctrl + Shift + G`：填充当前表单
- `Ctrl + Shift + H`：填充当前输入框

可在 `chrome://extensions/shortcuts` 中自定义。

### 3. 右键菜单

在网页中右键可看到以下菜单项：

- 填充所有输入框
- 填充当前表单
- 填充当前输入框

右键菜单可在扩展设置页中关闭。

## 可生成的数据类型

项目内置了一批更贴近中文使用场景的虚拟数据生成器，常见类型包括：

- 姓
- 名
- 姓名
- 用户名
- 邮箱
- 手机号
- 公司名称
- 地址
- 邮编
- 文本内容
- 数字
- 日期
- 时间
- 网址
- 密码
- 身份证号
- 正则生成内容
- 随机列表项

## 配置能力

扩展设置页支持以下能力：

- 调整字段匹配策略
- 配置忽略字段关键词
- 忽略隐藏字段
- 跳过已有值字段
- 控制是否触发输入/点击相关事件
- 配置固定密码或随机密码
- 维护自定义字段规则

自定义字段规则支持按关键词匹配字段，并指定填充类型，例如：

- 将包含 `phone`、`tel`、`手机` 的字段统一填充为手机号
- 将某类字段固定为随机数字范围
- 将某类字段限定为一组候选值中的随机项

## 目录结构

```text
.
├── public/                     # manifest 与扩展静态资源
├── src/
│   ├── background/            # background service worker
│   ├── content-script/        # 页面注入脚本
│   ├── common/                # 数据生成器与填充入口
│   ├── fill/                  # 原生控件与组件库控件适配
│   ├── options/               # 扩展设置页
│   ├── playground/            # 本地测试页面
│   ├── utils/                 # 存储等通用工具
│   └── types.ts               # 核心类型定义
├── vite.config.ts             # 主构建配置
├── vite.content.config.ts     # content script 构建配置
├── tsconfig.json
└── package.json
```


这部分适合在新增字段策略、修复交互事件、适配日期/选择类组件时做回归测试。

## 开发说明

### 核心执行链路

项目的大致执行流程如下：

1. `background` 接收工具栏点击、快捷键、右键菜单事件
2. `background` 向当前标签页发送填充消息
3. `content-script` 初始化 `ElementFiller`
4. `ElementFiller` 调用 `fill/orchestrator` 和各类控件适配器执行填充
5. 配置项通过 `chrome.storage.local` 持久化，并在修改后通知页面脚本刷新实例

### 适配实现

`src/fill/` 目录下按类型拆分：

- `native/`：原生控件填充
- `shared/`：事件触发、DOM 工具、浮层打开等共享逻辑
- `element-plus/`：Element Plus 组件适配
- `ant-design/`：Ant Design Vue 组件适配

如果你要新增组件库支持，通常从这里开始扩展。

## 常见使用场景

- 本地开发时快速填充注册、登录、资料编辑、地址信息等表单
- 测试环境中验证字段联动、表单校验和提交流程
- 为组件库表单封装编写适配逻辑并在 playground 中回归测试
- 演示系统时快速生成接近真实中文场景的数据

## 已知限制

- 部分站点的复杂自定义输入控件不会直接响应原生输入事件
- 某些表单字段命名不规范时，自动识别准确率会下降
- 组件库升级后，相关选择器或交互逻辑可能需要同步维护
- 当前为 Chrome 扩展实现，未针对其他浏览器做兼容性验证
